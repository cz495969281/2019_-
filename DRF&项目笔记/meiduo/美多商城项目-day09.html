美多商城项目

1. 购物车记录合并

当用户登录时，需要将cookie中的购物车数据和登录用户redis购物车中记录进行合并。

购物车记录合并不是一个新的API接口，只是在原来登录流程的基础上，补充合并的过程。

1）普通登录。

2）QQ登录。

怎么进行合并？(购物车记录合并方案)

假如id为2的用户登录之前，
cookie中的购物车数据如下:
    {
        11: {
            'count': 1,
            'selected': True
        },
        16: {
            'count': 2,
            'selected': False
        }
    }

redis中的购物车数据如下:
    cart_2: {
        '11': '2',
        '16': '1',
        '10': '1'
    }

    cart_selected_2: ('16', '10')

合并购物车记录时，

如果cookie中购物车数据和redis中购物车数据发生冲突，以cookie中的数据为准，拿cookie中的数据去覆盖redis中对应记录。

如果没有冲突，都保留。

cart_2: {
    '11': '1',
    '16': '2',
    '10': '1'
}

cart_selected_2: ('11', '10')
2. 订单结算商品信息

用户所要结算的商品的信息，就是redis购物车记录中被选中的商品。

API: GET /orders/settlement/
参数:
    前端通过请求头传递jwt token数据
响应:
    {
        'freight': '运费',
        'skus': [
            {
                'id': '商品id',
                'name': '商品名称',
                'price': '商品价格',
                'default_image_url': '默认图片',
                'count': '结算商品数量'
            },
            ...
        ]
    }
3. 订单创建(订单信息保存)
API: POST /orders/
参数:
    前端通过请求头传递jwt token数据
    {
        'address': '收货地址id',
        'pay_method': '支付方式'
    }
响应:
    {
        'order_id': '订单编号'
    }

订单信息保存基本流程:

1）向订单信息表中添加一条记录
2）订单中包含几个商品，就需要向订单商品表中添加几条记录
3）清除redis购物车中对应记录
