1. 订单事务

在进行订单信息保存时，数据库的操作要么都应该成功，要么都应该失败。

mysql事务: 一组mysql语句，要么都成功，要么都失败。

1）mysql事务的操作:

begin或start transaction: 
    手动开启一个mysql事务。
commit: 
    事务提交，让事务中sql语句操作结果永久生效。
rollback: 
    事务回滚，撤销事务中sql语句操作结果。

2）mysql事务保存点操作:

在事务中，可以设置事务的保存点，在进行事务回滚时，可以不回滚整个事务，而是回滚到指定的保存点。

savepoint 保存点名称:
    设置事务保存点。
rollback to 保存点名称:
    回滚到指定的保存点，保存点之后sql语句操作结果会被撤销。

3）Django中使用事务:

from django.db import transaction

with transaction.atomic():
    # with语句块中的代码，凡是涉及到数据库操作的语句，都会放在同一个事务中

    # 设置事务保存点
    sid = transaction.savepoint()

    # 写代码...

    # 回滚事务到指定的保存点
    transaction.savepoint_rollback(sid)
2. 订单并发

多个人同时购买同一件商品，可能会产生订单并发的问题。

订单并发问题解决：

1）悲观锁

​ 在事务中查询数据的时候尝试对数据进行加锁(互斥锁), 获取到锁的事务可以对数据进行操作，获取不到锁的事务会阻塞，直到锁被释放。

2）乐观'锁'

​ 乐观锁本质上不是加锁，查询数据的时候不加锁，对数据进行修改的时候需要进行判断,修改失败需要重新进行尝试。

3）celery任务队列

​ 把订单保存代码封装成celery任务函数，启动worker时只启动一个进程。

mysql事务隔离级别:

隔离级别 说明
Repeatable read 可重复读 在一个事务中执行查询语句时，获取的结果永远和第一次获取到结果一样，即使其他事务已经把数据修改并且进行了提交，当前事务仍然获取不到更新之后的结果。
Read committed 读取已提交 其他事务把数据修改并且提交之后，当前事务能获取到更新之后的结果。

注：Repeatable read 可重复读是mysql数据库默认隔离级别。

修改mysql事务的隔离级别:

1）编辑mysql数据库的配置文件

​ sudo vi /etc/mysql/mysql.conf.d/mysqld.cnf

2）修改配置项

 27 [mysqld]
 28 #
 29 # * Basic Settings
 30 #
 31 user        = mysql
 32 pid-file    = /var/run/mysqld/mysqld.pid
 33 socket      = /var/run/mysqld/mysqld.sock
 34 port        = 3306
 35 basedir     = /usr
 36 datadir     = /var/lib/mysql
 37 tmpdir      = /tmp
 38 lc-messages-dir = /usr/share/mysql
 39 skip-external-locking
 40 # 将mysql事务的隔离级别设置为读取已提交
 41 transaction-isolation=READ-COMMITTED

3）重启mysql服务

​ sudo service mysql restart

3. 支付宝接入

线上环境：

1）登录支付宝开发平台。

2）创建开发者应用，设置相关配置信息，提交并等待审核。

3）审核通过，应用上线，完成签约。

沙箱环境：对真实支付宝环境模拟，让开发者在不创建线上应用情况下就可以使用沙箱环境进行开发。

SDK(软件开发工具包)：对原生API进行封装。

python支付宝SKD: pip install python-alipay-sdk --upgrade

4. 支付API接口

1）获取支付宝支付网址API

API: GET /orders/(?P<order_id>\d+)/payment/
参数:
    在请求头中传递jwt token数据
    在url地址中传递订单id
响应:
    返回支付宝支付网址
    {
        'alipay_url': '支付网址'
    }

2）保存支付结果API

API: PUT /payment/status/?支付宝参数
参数:
    查询字符串传递支付宝参数
响应:
    {
        'trader_id': '支付宝交易号'
    }
