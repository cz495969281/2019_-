美多商城项目

1. 用户中心-最近浏览

1）什么时候需要保存用户的最近浏览记录？

答：当用户浏览商品的详情页面时，需要保存浏览记录。

2）浏览记录保存在哪里？redis

答：

​ mysql: 用户频繁浏览商品，需要频繁操作数据库。

​ redis: 从redis中存取数据，效率要比mysql要快得多。

3）存储浏览记录时，需要保存哪些数据？

答：存储用户浏览的商品id。

4）采用redis中的哪种数据格式进行存储？

答：list

# string: 字符串 <key>: <value>
    history_<user_id>: '1,2,3'

# hash: 哈希  <key>: { <field>:<value>, <field>: <value>}
    history: {
        history_<user_id>: '1,2,3',
        ...
    }

# list: 列表  <key>: [<value>, <value>, ...]
    history_<user_id>: [<sku_id>, <sku_id>, ...]

# set: 无序集合   <key>: (<value>, <value>, ...)
    pass

# zset: 有序集合  <key>: (<value>-权重，<value>-权重, ...)
    zset保持元素的顺序，每个元素都有对应权重值

5）历史浏览记录添加的过程？

答:

假如id为2的用户的浏览记录如下:
    history_2: ['5', '2', '4', '3',  '1']

去重：如果用户已经浏览过某个商品，需要从redis列表中先移除对应商品id.
保持顺序: 左侧加入。
截取: 只保留用户最新的几个浏览记录。

5. 商品搜索

需求: 根据商品名称(name)或者商品副标题(caption)搜索商品的信息。

假设用户输入的关键字: iPhone

select * from tb_sku where name like '%iPhone%' or caption like '%iPhone%';

对于数据库查询语句，like语句查询效率很低。

搜索引擎：

​ 搜索引擎可以对数据表的中进行处理，建立一份索引数据(记录了索引字段和表数据之间对应关系)，在建立索引数据时，会对索引字段的内容进行关键词的拆分，记录每个关键词在哪些索引数据中存在。

例如: es slor whoosh

索引字段:

​ 根据什么内容搜索信息，这些内容就是索引字段。

全文检索框架: haystack

​ 1）帮助开发者使用搜索引擎的功能(建立索引，根据关键词搜索索引数据)

​ 2）根据索引数据去查询数据库中对应数据。
